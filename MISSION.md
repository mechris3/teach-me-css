# Mission: CSS Mastery

## Why
Go from a senior frontend developer who frequently Googles CSS to someone with deep, near-exhaustive knowledge — particularly of modern CSS features. The goal is fluency: being able to reach for the right CSS solution from memory, without needing to look it up, and understanding the *why* behind how things work.

## Success looks like
- Reaching for custom properties, modern functions (`clamp()`, `min()`, `max()`, `color-mix()`, etc.) instinctively
- Writing custom CSS functions (`@function`) confidently
- Using the `attr()` function and data attributes for dynamic styling
- Building complex layouts with Flexbox and Grid from memory, including edge cases
- Creating polished animations and transitions without reference material
- Understanding the cascade, specificity, and inheritance at a deep enough level to debug anything
- Knowing what's new and cool in CSS — container queries, scroll-driven animations, anchor positioning, `@layer`, nesting, `@scope`, `light-dark()`, and whatever else is landing

## Constraints
- Already a 20+ year frontend developer specialising in Angular — not starting from zero
- Lessons should respect existing knowledge and skip fundamentals
- Focus on depth and retention, not breadth of basics already known

## Out of scope
- HTML fundamentals
- JavaScript fundamentals
- CSS preprocessors (Sass/Less) — focus on native CSS
- Framework-specific styling (Angular component styles, encapsulation) — unless directly relevant to a CSS concept
