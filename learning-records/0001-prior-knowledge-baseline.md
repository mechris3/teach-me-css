# Prior knowledge: Senior frontend developer with 20+ years experience

The user is a senior frontend developer with 20+ years of experience, currently specialising in Angular. They are very familiar with JavaScript, HTML, and CSS at a working level — but self-describe as someone who "often needs to Google CSS stuff." This means the foundational mental model is solid but retrieval strength (storage strength) is low for many properties and patterns. Teaching should target storage strength, not introduction.

## Evidence
User disclosed in initial session.

## Implications
- Skip all beginner material. The user knows the box model, selectors, positioning, floats (historically), media queries, etc.
- Focus on areas they specifically flagged: custom properties (variables), CSS functions, animations, transitions, Flexbox, Grid.
- Lessons should be dense and fast-moving. The user doesn't need hand-holding on concepts — they need structured practice to move knowledge from "I can Google it" to "I know it cold."
- Retrieval practice and interleaving are the primary tools here. The user already has the knowledge *somewhere* — we need to make it stick.
