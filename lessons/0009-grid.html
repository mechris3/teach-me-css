CSS Grid — Two-Dimensional Layout

Flexbox handles one axis at a time. Grid handles both axes simultaneously — you define columns AND rows up front, and items flow into cells. It's designed for page-level layouts and complex component structures where you need control over both dimensions.

The mental model

Grid introduces a few concepts that don't exist in Flexbox:

  • Lines — the invisible dividers between tracks. A 3-column grid has 4 column lines (numbered 1–4).
  • Tracks — the rows or columns themselves (the space between two lines).
  • Cells — the intersection of one row track and one column track.
  • Areas — a rectangular region spanning one or more cells.

Visual diagram: a 3-column × 2-row grid with all entities labelled

Key difference from Flexbox: In Flexbox, the container distributes space and items just flow. In Grid, the container defines a structure (rows + columns) and items are placed into it — either automatically or explicitly.

The fr unit — fractional space

fr means "fraction of the remaining space" — it's Grid's equivalent of flex-grow. After fixed-size tracks are laid out, the remaining space is divided proportionally among fr tracks:

1fr 2fr 1fr — middle column gets double the space

fr vs %: Percentages are based on the container width and don't account for gaps. fr divides only the remaining space after gaps and fixed tracks are subtracted. Use fr in grid — it's almost always what you want.

auto-fill vs auto-fit: Both create as many tracks as fit. The difference: auto-fill keeps empty tracks (space stays reserved). auto-fit collapses empty tracks so existing items expand to fill. For most card grids, auto-fit is what you want.

Placing items explicitly — grid lines

By default, items auto-flow into the next available cell. But you can place items precisely using line numbers:

Item 1 spans columns 1–3, Item 2 spans rows 2–4

Cols 1–3
Rows 2–4
Auto
Auto
Auto
Auto
.header {
  grid-column: 1 / 3;  /* start at line 1, end at line 3 (span 2 columns) */
  grid-row: 1;         /* just row 1 */
}

.sidebar {
  grid-row: 2 / 4;     /* span rows 2 and 3 */
}

/* Shorthand with span keyword: */
.wide { grid-column: span 2; }     /* span 2 columns from wherever placed */
.tall { grid-row: span 3; }        /* span 3 rows */

/* Negative lines count from the end: */
.full-width { grid-column: 1 / -1; }  /* first line to last line = full width */

Line numbering: Lines are numbered from 1, not 0. A 3-column grid has column lines 1, 2, 3, 4. The notation 1 / 3 means "from line 1 to line 3" — which spans 2 tracks. It's a half-open range: the start line is included, the end line is excluded (the item stops before it).

Named template areas — the most readable layout

For page layouts, template areas let you draw the layout as ASCII art:

.layout {
  display: grid;
  grid-template-columns: 150px 1fr;
  grid-template-rows: auto 1fr auto;

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }

Why this is powerful: You can completely rearrange the layout by changing the template-areas string — no touching the item CSS. Use a media query to redefine the area map for mobile, and the whole layout restructures.

The implicit grid — what happens beyond your template

When items overflow your defined template (more items than cells), Grid creates implicit tracks automatically. You control their size with:

.container {
  grid-template-columns: 1fr 1fr;   /* explicit: 2 columns */
  grid-auto-rows: 60px;              /* implicit rows: 60px each */
  /* grid-auto-rows: minmax(60px, auto);  — at least 60px, grow with content */
}

Alignment in Grid — same axes, same properties

The justify/align mental model from Flexbox carries directly into Grid:

  • justify-* → inline axis (horizontal in normal writing mode)
  • align-* → block axis (vertical)

But Grid adds more granularity:

Property What it aligns Axis
justify-items All items within their cells Inline (horizontal)
align-items All items within their cells Block (vertical)
justify-content The entire grid within the container Inline (horizontal)
align-content The entire grid within the container Block (vertical)
justify-self One item within its cell Inline (horizontal)
align-self One item within its cell Block (vertical)

The pattern: *-items = set a default for all items. *-self = override for one item. *-content = position the grid itself when it's smaller than its container. Same logic as Flexbox, just more options because Grid has cells to position items within.

place-items: center — centers every item in its cell (shorthand for align-items + justify-items)

minmax() — flexible tracks with bounds

minmax(min, max) defines a track that's at least min and at most max. Pairs naturally with auto and fr:

minmax(100px, 1fr) — columns are at least 100px, grow to fill equally

grid-template-columns: repeat(3, minmax(100px, 1fr));

/* At least content size, at most 300px */
grid-template-rows: minmax(auto, 300px);

/* The responsive grid pattern: */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

Subgrid — inheriting the parent's tracks

A grid item can itself be a grid container. But by default, its inner grid is independent — it doesn't align with the parent's columns/rows. subgrid fixes this:

Inner grid inherits the parent's column tracks via subgrid

1
2
3
Sub 1
Sub 2
Sub 3
.parent {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
}

.child-spanning-all-columns {
  grid-column: 1 / -1;
  display: grid;
  grid-template-columns: subgrid;  /* use parent's column tracks */
}

When to use subgrid: When a nested component needs to align with the parent's grid lines. Common cases: card headers aligning across a row of cards, form labels aligning across nested fieldsets, table-like layouts with grouped rows.

Grid vs Flexbox — when to use which

Use Flexbox when... Use Grid when...
Content drives layout (items determine size) Layout drives content (structure defined first)
One-dimensional (row OR column) Two-dimensional (rows AND columns)
Items should wrap naturally Items should align to a defined structure
Nav bars, button groups, simple card rows Page layouts, dashboards, complex component grids
Unknown number of items Known structure, items placed precisely

They're not competitors. A page layout might use Grid at the top level (header/sidebar/main/footer) with Flexbox inside components (nav items, card contents, button rows). Use both — choose based on whether you're distributing in one dimension or placing in two.
